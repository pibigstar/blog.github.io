1 ReentrantLock

它是一个可重入锁，即一个线程可重复加锁，即当前线程获取该锁再次获取不会被阻塞。获得几次锁，解锁也要解几次。
它有两种实现，公平锁和非公平锁。

  • 公平锁：锁的获取顺序就应该符合请求上的绝对时间顺序
  • 非公平锁：不按请求的时间，谁抢到就是谁的

默认实现的为非公平锁，因为公平锁为了保证时间上的绝对顺序，需要频繁的上下文切换，而非公平锁会降低一定的上下文切换，降低性能开销。
